Objetivo da Etapa:

Nesta primeira etapa, os alunos desenvolverão uma aplicação monolítica simples utilizando Spring Boot, preparando a base para futuras evoluções em direção a uma arquitetura de microsserviços.

Subcompetências a serem Desenvolvidas:

Implementação de uma Aplicação com Spring Boot:

Desenvolver uma aplicação seguindo um design em camadas (controle, serviço e repositório).
Utilizar a Autoconfiguração do Spring Boot para simplificar o setup inicial do projeto.

Gerenciamento de Dependências:

Aprender a gerenciar dependências de forma eficaz, preparando o projeto para escalabilidade e manutenção.

Desenvolvimento de APIs REST:

Implementar APIs REST utilizando as anotações do Spring MVC para facilitar a comunicação entre o front-end e o back-end.

Práticas de Código Limpo e Manutenção:

Aplicar princípios SOLID e padrões de design para garantir que o código seja limpo e fácil de manter.

Modelagem de Domínios:

Modelar casos de negócio usando conceitos de domínios, subdomínios e bounded contexts do Domain-Driven Design (DDD).

Desenvolvimento Front-End:

Criar uma interface de usuário básica usando React para consumir informações do back-end, facilitando a interação do usuário com a aplicação.

Entregas Esperadas:

Código Fonte:

Repositório Git com o código fonte da aplicação, incluindo front-end e back-end.

Documentação:

Documento explicando a arquitetura da solução, incluindo diagramas de componentes e sequência que ilustrem o design da aplicação.

Demonstração:

Uma apresentação prática da aplicação em funcionamento, demonstrando as funcionalidades implementadas e a interação entre front-end e back-end.
